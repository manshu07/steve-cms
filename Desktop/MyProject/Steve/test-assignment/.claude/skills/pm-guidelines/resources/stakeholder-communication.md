# Stakeholder Communication

## Communication Types

### Weekly Status Update
```markdown
# Weekly Status - Week of [Date]

## Summary
One sentence summary of key progress.

## Accomplishments
- Shipped feature X to production
- Completed Y% of feature Z
- Resolved critical bug W

## Next Week Focus
- Complete feature Z
- Start feature A
- Address technical debt B

## Risks & Blockers
- Risk 1: Mitigation plan
- Blocker 1: Need decision from stakeholder

## Metrics
| Metric | Current | Target | Trend |
|--------|---------|--------|-------|
| Sprint completion | 85% | 90% | ↑ |
| Bug count | 12 | <10 | ↓ |
```

### Release Announcement
```markdown
# Release [Version] - [Feature Name]

## What's New
- Feature 1: Description
- Feature 2: Description

## Bug Fixes
- Fixed issue with X
- Resolved Y problem

## Known Issues
- Issue A: Workaround is X

## Timeline
- Staged rollout: [Date]
- Full release: [Date]

## Action Required
- Update integration by [Date]
- Review new documentation
```

## Meeting Types

### Kickoff Meeting
**Duration:** 1 hour
**Attendees:** Stakeholders, team leads

**Agenda:**
1. Project overview (10 min)
2. Goals and success metrics (10 min)
3. Timeline and milestones (15 min)
4. Q&A (15 min)
5. Next steps (10 min)

### Demo Meeting
**Duration:** 30 minutes
**Frequency:** End of sprint

**Format:**
1. Context (2 min)
2. Demo (20 min)
3. Feedback (8 min)

```markdown
## Impact

## Options
| Option | Pros | Cons | Recommendation |
|--------|------|------|----------------|
| A | ... | ... | ✓ |

## Decision Needed By

## Owner
[Name] for follow-up
```
