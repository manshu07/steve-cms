## X/Twitter Templates

### Template 1: Data Story

```
[Headline with number]

[Context sentence]

[Supporting detail]

[Proof/metric]

[Insight/takeaway]
```

**Example:**
```
We increased trial-to-paid by 40% with one email.

Tried sending onboarding tips at the right moment.

Instead of day 1, we send on day 3.
After users have hit a blocker.

Result: 40% more conversions.

Timing beats optimization.
```

---

### Template 2: Thread (Story)

```
Tweet 1: Hook + result

Tweet 2: The problem (context)

Tweet 3: The attempt (what didn't work)

Tweet 4: The breakthrough (what worked)

Tweet 5: The result (numbers)

Tweet 6: The lesson (one sentence)
```

**Example:**
```
(1/6) We went from 0 to 10k users in 30 days. Here's how:

(2/6) Launched on PH. Got 400 upvotes.
Almost no signups.

(3/6) Tried cold email. Sent 1,000.
7 replies. 2 signups. Waste of time.

(4/6) Changed strategy.
Created content showing the problem.
And how we solve it.

(5/6) Posted on HN. Explained the tech.
Got 500 signups in 24 hours.

(6/6) The lesson:
Build in public → people pay attention.
Launch when ready → crickets.
```

## LinkedIn Templates

### Template 1: Problem-Solution-Result

```
[Strong hook - the painful problem]

[Context - why this matters]

[The wrong approach - what most do]

[The right approach - what actually works]

[Results - numbers, proof]

[Key takeaways bullet points]

[Discussion question]
```

**Example:**
```
Most founders optimize for vanity metrics.

We did too. Then we almost ran out of cash.

The trap:
We focused on user growth.
Registered users looked great.
Revenue didn't move.

The reality:
Users who don't pay... aren't users. They're leads.

So we changed:
1. Stopped counting free users as "growth"
2. Focused on activation rate (free → paid)
3. Killed features that free users loved but wouldn't pay for

The result:
We have fewer users.
But 3x the revenue.

Key takeaways:
• Growth without revenue is just burning cash
• Paying customers are the only users that matter
• Sometimes you have to shrink to grow

Has anyone else had to make this hard choice?
```

---

### Template 2: Leadership/Culture

```
[Observation about culture/hiring]

[Specific story/example]

[What we learned]

[Practical advice]

[Question for audience]
```

**Example:**
```
I stopped hiring "culture fit" people.

Here's what happened instead:

Last year, our team was homogeneous.
Same background. Same schools. Same thinking.

We moved fast. But we missed things.

Big things.

So we changed our hiring:
- No more "culture fit" interviews
- Started hiring "culture add"
- Looked for people who disagreed with us

Result:
Team is slower initially.
But the ideas are better.
We catch problems we never saw before.

Practical advice:
Hire 10% of people who make you uncomfortable.
They'll see what you're missing.

Who's the best "culture add" you've made?
```

### Template 3: Behind the Scenes

```
[Hook - something people don't see]

[The reality vs. appearance]

[Specific numbers/details]

[What we learned]

[Question]
```

**Example:**
```
Behind the scenes of our "overnight success":

Social media made it look like:
• We launched → blew up → profit

Reality:
• 2 years of building in public
• 500+ cold emails sent
• 3 failed launches before this one

The numbers nobody talks about:

• 47 podcast appearances
• 18 blog posts that got <100 views
• 3 pivots before finding product-market fit

Success looks fast from the outside.
From the inside, it's just persistence.

What's something you wish people knew about your journey?
```

## Newsletter Templates

### Template 1: One Big Insight

```
[HEADER: WEEKLY #XX]

🔥 [Strong headline]

[Opening paragraph with the story]

[The problem - context]

[The solution - what we did]

[The results - numbers]

[The lesson - one sentence]

---

📊 This week
• [Update 1]
• [Update 2]
• [Update 3]

🎯 One cool thing
[Link + one sentence]

[CTA]
```

**Example:**
```
WEEKLY #47

🔥 The 15-minute fix that saved us $7,440/month

Our AWS bill was $12,400 per month.
We're a 5-person startup. That's... a lot.

Looked at the bill:
• EC2 instances: $9,200
• Everything else: $3,200

The problem:
Dev team spins up servers for batch jobs.
Forgets to turn them off.

They're running for days. Weeks. Months.

The fix:
A 15-line cron job.
Runs at 6pm every day.
Kills orphaned EC2 instances.

The result:
Next month's bill: $4,960
Savings: $7,440/month
ROI: 49,600% on 15 minutes of work

The lesson:
Expensive problems often have cheap fixes.
You just have to look.

---

📊 This week
• Launched cost monitoring dashboard
• Added infrastructure review to sprint planning
• Set up auto-shutdown for dev environments

🎯 One cool thing
[Vantage.sh - AWS cost monitoring with 1-line setup]

What's your most expensive AWS mistake?
Hit reply—I'd love to hear it.
```

## Hook Library

### Data Hooks
```
"We [metric] by [number]%"
"[Number] things I learned from [experience]"
"The [number]-week experiment that [result]"
"[Number]% of [group] are doing [wrong thing]"
```

### Story Hooks
```
"We spent [amount] on [thing]. Here's what we got."
"Tried [approach] for [time]. [Result]."
"[Company] did [thing]. Here's the breakdown."
"The [time period] that changed everything."
```

### Contrarian Hooks
```
"Most [people/companies] [wrong thing]. Here's why."
"[Popular thing] is broken. Use this instead."
"Stop [bad thing]. Start [good thing]."
"Unpopular opinion: [surprising take]."
```

### Teaching Hooks
```
"How to [outcome] in [timeframe]"
"The [number]-step framework for [result]"
"Stop [mistake]. Start [solution]."
"The only [tool/strategy] you need for [goal]."
```

### Curiosity Hooks
```
"The [number]-word email that [result]"
"Why we [surprising decision]"
"The mistake that cost us [amount]"
"What nobody tells you about [topic]"
```

## Quality Checklist by Platform

### X
- [ ] Under 280 chars per tweet
- [ ] Strong first line
- [ ] One clear idea
- [ ] No hashtag spam
- [ ] Link only if necessary

### LinkedIn
- [ ] Strong opening hook
- [ ] Short paragraphs (2-3 sentences)
- [ ] Bullet points for takeaways
- [ ] Discussion question or CTA
- [ ] Professional but conversational

### TikTok/Short Video
- [ ] Visual hook in first 3 seconds
- [ ] Script around visuals
- [ ] One demo, one claim, one CTA
- [ ] Under 60 seconds
- [ ] Clear CTA at end

### YouTube
- [ ] Result shown in first 30 seconds
- [ ] Chapter timestamps in description
- [ ] Visuals change every 20-30 seconds
- [ ] 8-15 minutes for tutorials
- [ ] Subscribe reminder + link

### Newsletter
- [ ] One clear insight
- [ ] Skimmable sections
- [ ] Opening paragraph does work
- [ ] Clear CTA
- [ ] Mobile-friendly formatting
