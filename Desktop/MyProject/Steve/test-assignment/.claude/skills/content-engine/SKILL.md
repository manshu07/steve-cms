# Content Engine Skill

Turn one idea into strong, platform-native content instead of posting the same thing everywhere.

## Purpose
Transform a single source asset (article, video, demo, launch, milestone) into platform-specific content that reads natively on each platform rather than cross-posting the same copy everywhere.

## When to Use This Skill

**Auto-activates when:**
- Writing X posts or threads
- Drafting LinkedIn posts or launch updates
- Scripting short-form video (TikTok, Reels, Shorts)
- Creating YouTube explainer scripts
- Repurposing articles, podcasts, demos, or docs into social content
- Building content plans around launches, milestones, or themes
- Creating content calendars or multi-platform campaigns

**Keywords that trigger:**
- social post, tweet, X post, LinkedIn, thread
- TikTok, Reels, Shorts, video script
- YouTube, content, content calendar
- repurpose, cross-post, multi-platform
- social media, launch content, campaign
- newsletter, social strategy

**2. Audience**

- Customers/users?
- Operators/founders?
- General audience?

**3. Platform(s)**
- X, LinkedIn, TikTok, YouTube, Newsletter?
- Single platform or multi-platform campaign?

**4. Goal**
- Awareness (get seen)

- Recruiting (hiring)
- Authority (thought leadership)
- Launch support (announce something)
- Engagement (discussion)

## Core Rules

### ✅ DO These Things

1. **Adapt for the platform** - Each platform has its own language
2. **Hooks matter more than summaries** - Lead with the interesting thing
3. **One clear idea per post** - Don't cram everything in
4. **Use specifics over slogans** - Numbers, examples, details
5. **Keep the ask small and clear** - One CTA, not five

### ❌ NEVER Do These Things

1. **Cross-post identical copy** - X ≠ LinkedIn ≠ Newsletter
2. **Generic hooks** - "Excited to announce..." or "Hot take:"
3. **Multiple ideas in one post** - One thread/post = one concept
4. **Hashtag spam** - 2-3 relevant tags max
5. **Vague CTAs** - "Check it out" vs "Link in bio → get started free"

## Platform Guidance

### X (Twitter)

**Characteristics:**
- Open fast (first 2 lines matter most)
- One idea per post OR per tweet in a thread
- Keep links out of main body unless necessary
- Threads = one argument, multiple tweets
- Avoid hashtag spam (0-3 relevant tags)

**Structure:**
```
Hook (first line)
Supporting point
Evidence/example
CTA (or continue in thread)

[Optional: Thread continues]
```

**Example:**
```
We cut AWS costs by 60% by counting servers.

Dev team spun up 47 instances for a batch job.
Left them running. For 3 months.

Cost: $12,400.

Fixed it with a cron job that kills orphans at 6pm.
Took 15 minutes to write.

Lesson: Expensive problems often have cheap fixes.
```

### LinkedIn

**Characteristics:**
- Strong first line (hook matters)
- Short paragraphs (2-3 sentences max)
- More explicit framing around lessons, results, takeaways
- Professional but conversational tone
- Visuals matter (screenshots, diagrams, charts)

**Structure:**
```
Strong hook (first line, clear value)

The problem (what broke)
The approach (what we did)
The result (numbers, proof)

Key takeaways:
• Point 1
• Point 2
• Point 3

CTA or discussion question
```

**Example:**
```
Most SaaS companies die from success, not failure.

Here's the trap that killed our growth:

We landed a $50k customer.
They wanted custom features.
We built them.

Then they churned.
And we were stuck maintaining custom code nobody else wanted.

The math was brutal:
• Custom feature: 2 weeks to build
• One customer: pays $50k/year
• Maintenance: $30k/year forever
• Breakeven: Need 3 more customers

The fix:
We stopped doing custom features.
Now we build for the market, not the customer.

If one company needs it, others probably do too.

Has anyone else fallen into this trap?
```

### TikTok / Short Video (Reels, Shorts)

**Characteristics:**
- First 3 seconds MUST interrupt attention
- Script around visuals, not just narration
- One demo, one claim, one CTA
- 15-60 seconds optimal
- Hook → Problem → Solution → Proof → CTA

**Structure:**
```
[0-3s] Visual hook + problem
[3-10s] Show the issue (demonstrate pain)
[10-30s] The solution (demo the fix)
[30-50s] Result/proof (it worked)
[50-60s] CTA (try it, link in bio)
```

**Script Example:**
```
[VISUAL: Screen recording of slow dashboard]
HOST (off-screen): "This dashboard takes 8 seconds to load."

[VISUAL: Loading spinner, frustration]
HOST: "Users were bouncing. We were losing sales."

[VISUAL: Fast scroll through code]
HOST: "Added Redis caching. 47 lines of code."

[VISUAL: Side-by-side: before vs after]
HOST: "Now it loads in 0.3 seconds. 27x faster."

[VISUAL: Graph showing traffic increase]
HOST: "Bounce rate dropped 40%. Sales up 22%."

[VISUAL: Final screen with URL]
HOST: "Link in bio for the full breakdown."
```

### YouTube

**Characteristics:**
- Show the result early (first 30 seconds)
- Structure by chapter (timestamp in description)
- Refresh visual every 20-30 seconds
- 8-15 minutes for tutorials, 2-5 minutes for updates
- Hook → Context → Solution → Demo → Result

**Structure:**
```
[0:00] Hook (the result, not the topic)
[0:30] Context (why this matters)
[2:00] The approach (what we did)
[5:00] Demo (show the work)
[8:00] Results (numbers, proof)
[10:00] Key takeaways
[11:00] CTA (subscribe, link below)
```

**Script Example:**
```
[VISUAL: Final result on screen]
HOST: "We built an AI that writes sales emails.

It booked 127 meetings last month.
Here's how we built it."

[VISUAL: Team struggling with emails]
HOST: "Our sales team was drowning.
4 hours a day on research.
Low reply rates.
Burnout."

[VISUAL: Architecture diagram]
HOST: "The solution:
1. Scrape prospect data (LinkedIn, company site)
2. GPT-4 writes personalized emails
3. Sends at optimal times (AI learns)
4. Updates CRM automatically"

[VISUAL: Screen recording of the tool]
HOST: "Let me show you how it works..."

[DEMO: Walk through the tool]

[VISUAL: Metrics dashboard]
HOST: "The results:
• 28% reply rate (vs 8% before)
• 3.2x more meetings booked
• 67% less time on research"

[VISUAL: Head shot]
HOST: "Link below to try it free.

Like and subscribe if you want more builder content."
```

### Newsletter

**Characteristics:**
- Deliver one clear lens, not unrelated items
- Make section titles skimmable
- Opening paragraph does real work (sets frame)
- Mix insight with updates, not diary filler
- Clear CTA (reply, click, or no action)

**Structure:**
```
Above the fold (strong, skimmable)
One big insight (the main value)
Quick updates (bullets, not prose)
One cool thing (interesting find)
Ask or CTA
```

**Example:**
```
WEEKLY #47

🔥 We cut AWS costs by 60% with a 15-line script

[Opening paragraph with the story]

The expensive mistake
Our dev team spun up 47 EC2 instances.
Left them running.
Cost: $12,400.

The fix
A cron job that kills orphans at 6pm.
Took 15 minutes to write.

The lesson
Expensive problems often have cheap fixes.

• Launched the new dashboard

[link to interesting article/tool]
```

## Repurposing Flow

### Default Cascade

**Step 1: Identify Anchor Asset**
- Article, video, podcast, demo, launch doc, memo
- Extract the core value/insight

**Step 2: Extract Atomic Ideas**
- Pull 3-7 standalone concepts from the source
- Each idea should work independently
- Rank by impact/sharability

**Step 3: Write Platform-Native Variants**
- Adapt each idea for each target platform
- Use platform-specific formats and hooks
- Match platform tone and style

**Step 4: Trim Repetition**
- Ensure variety across outputs
- Don't say the same thing twice on the same day
- Space out related posts

**Step 5: Align CTAs**
- Match CTA to platform intent
- X: "Comment below" or "Read more"
- LinkedIn: "Share your thoughts" or "DM me"
- Video: "Link in bio" or "Subscribe"
- Newsletter: "Hit reply" or "Click here"

## Campaign Deliverables

When asked to create a campaign, return:

### 1. Core Angle
- What's the main message?
- Why does it matter now?
- What's the proof/credibility?

### 2. Platform-Specific Drafts
- X posts (single or threads)
- LinkedIn posts
- Video scripts (if applicable)
- Newsletter issue (if applicable)

### 3. Posting Order (optional)
- What posts first?
- What reinforces what?
- Timing considerations

### 4. CTA Variants (optional)
- Different CTAs for different platforms
- A/B test options

### 5. Missing Inputs (if any)
- Metrics/numbers needed
- Screenshots/visuals required
- Links to include
- Legal/compliance checks

## Quality Gate

Before delivering any content, verify:

### Platform Native Check
- [ ] X post reads like a native X post (not LinkedIn)
- [ ] LinkedIn post reads like LinkedIn (not X)
- [ ] Video script uses visual-first structure
- [ ] Newsletter uses skimmable sections

### Hook Quality
- [ ] First line hooks attention
- [ ] Specific, not generic
- [ ] Creates curiosity or tension

### Content Quality
- [ ] One clear idea per post
- [ ] Specifics over slogans
- [ ] Numbers/examples included
- [ ] No generic hype language

### CTA Quality
- [ ] Clear and specific
- [ ] Matches platform intent
- [ ] Appropriate ask for audience

- [ ] No duplicated copy across platforms (unless requested)

## Common Mistakes to Avoid

### Cross-Platform Mistakes

**❌ Posting same copy everywhere**
```
X: "We're excited to launch our new AI tool! #AI #launch"
LinkedIn: "We're excited to launch our new AI tool! #AI #launch"

✅ BETTER:
X: "Built an AI that writes sales emails. Booked 127 meetings."
LinkedIn: Full story with context and results
```

**❌ Using hashtags on every word**
```
❌ "We launched #AI #SaaS #B2B #sales #tool"
✅ "We launched an AI tool for sales teams"
```

**❌ Generic hooks**
```
❌ "Hot take: Pricing matters"
✅ "We doubled revenue by raising prices. Here's what happened."
```

## Hook Formula Library

### Data-Based Hooks
- "We [metric] by [number]%"
- "[Number] things I learned from [experience]"
- "The [number]-week experiment that changed [outcome]"

### Contrarian Hooks
- "Most [people/companies] [wrong thing]. Here's the truth."
- "[Popular thing] is broken. Here's what works."
- "Unpopular opinion: [surprising take]"

### Story-Based Hooks
- "We spent [amount] on [thing]. Here's what we got."
- "Tried [approach] for [time period]. [Result]."
- "[Person/Company] did [thing]. Here's the breakdown."

### Teaching Hooks
- "How to [outcome] in [timeframe]"
- "The [number]-step framework for [result]"
- "Stop [bad thing]. Start [good thing]."
