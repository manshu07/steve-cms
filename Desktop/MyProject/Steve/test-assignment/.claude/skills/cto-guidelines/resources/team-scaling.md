# Team Scaling

## Team Structures

### Startup (1-10 engineers)
```
CTO
├── Backend Team
├── Frontend Team
└── DevOps (shared)
```

### Growth (10-30 engineers)
```
CTO
├── Platform Squad
│   ├── Backend (3-4)
│   └── DevOps (2)
├── Product Squad A
│   ├── Full-stack (4)
│   └── QA (1)
└── Product Squad B
    ├── Full-stack (4)
    └── QA (1)
```

### Scale-up (30-100 engineers)
```
CTO
├── VP Engineering
│   ├── Tribe: Platform
│   ├── Tribe: Growth
│   └── Tribe: Core Product
└── VP Architecture
    ├── Security
    └── SRE
```

### Interview Process
1. **Resume Screen (30 min)** - Recruiter
2. **Technical Phone (60 min)** - Engineer
3. **Onsite (4 hours)**
   - Coding exercise (90 min)
   - System design (60 min)
   - Behavioral (45 min)
   - Team fit (45 min)
4. **Decision Meeting** - All interviewers

### Interview Rubric
```markdown
| Criteria | Strong No | No | Neutral | Yes | Strong Yes |
|----------|-----------|----|---------|----|-----------|
| Technical | 1 | 2 | 3 | 4 | 5 |
| Communication | 1 | 2 | 3 | 4 | 5 |
| Problem Solving | 1 | 2 | 3 | 4 | 5 |
| Culture Fit | 1 | 2 | 3 | 4 | 5 |
```

## Onboarding

### Week 1
- Company and team introductions
- Development environment setup
- Codebase overview
- First small task

### Week 2-4
- Paired programming sessions
- Code reviews
- Feature ownership
- Documentation review

### 30-60-90 Day Goals
```markdown
## 30 Days
- Complete onboarding tasks
- Ship first feature
- Meet all team members

## 60 Days
- Own a service/feature
- Complete code reviews independently
- Contribute to team discussions

## 90 Days
- Lead a project
- Mentor newer team members
- Propose improvements
```

## Team Health

### Health Check Questions
1. Is our process working?
2. Are we delivering value?
3. Is communication effective?
4. Are we learning and growing?
5. Is there psychological safety?
